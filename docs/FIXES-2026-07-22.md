## What changed

1. **/review is now private** (main.py). REVIEW_TOKEN in .env → open
   /review?token=<value> (or Authorization: Bearer). Unset = localhost only.
   filename/error are html.escape()d (stored-XSS fix).
2. **Rate limiting works behind Cloudflare Tunnel** (rate_limit.py, main.py).
   TRUST_PROXY=1 in .env → limiter keys on CF-Connecting-IP / first
   X-Forwarded-For hop instead of the tunnel's 127.0.0.1 (which is exempt).
   Headers are ignored unless TRUST_PROXY=1 (spoof protection).
3. **Upload size caps everywhere** — the STORE_UPLOADS corpus read now uses
   the capped chunked reader (was: unbounded RAM buffer), and
   /visualize/render + /visualize/animate reject >MAX_RENDER_UPLOAD_MB
   (default 15 MB) with 413 and empty files with 400.
4. **One GPU, one queue** (new server/gpu_gate.py). Parse-ML inference and
   SDXL/SVD renders now share a single semaphore (MAX_CONCURRENT_GPU, default
   1) — a render and a parse can no longer co-run into a 12 GB OOM.
5. **RERA super built-up corrected** (area_statement.py): super = CARPET x
   loading (market convention), floored at built-up. Was built_up x loading
   (double-counted walls; showed ~68% efficiency where ~77% is right). Note
   added to the statement; 2 new pinned tests.
6. **BOQ label honest** (App.tsx insights card): "Walls & finishes cost" +
   explicit exclusions line (RCC, doors/windows, MEP). Math unchanged.
7. **tf2 reader revived** (tf2_floorplan.py): it could never emit a door
   (0/1 mask vs door=pixel-value-2), so plan_health's no_doors -100 made it
   lose every best-of. Openings are now typed by geometry: outer-edge ->
   window, interior -> door (mirrors scene_builder's 6% edge rule; general,
   not per-plan). 3 new synthetic-mask tests.
8. **License gates** (.env flags, no code changes needed to flip):
   DISABLE_CUBICASA (CC BY-NC — non-commercial only), DISABLE_TF2 (GPL-3.0 —
   NOT "commercial-safe", header comment corrected), DISABLE_SVD (Stability
   community license; /visualize/animate → 403 when set).
9. **BUILD FIX**: deb56aa did not compile — LandingHero (frozen) passes
   onEnterApp to AppSections, which lost that prop in the ContactModal
   refactor (TS2322). Restored as an optional prop on AppSections (landing
   page untouched, rule 4); "back to the viewer" now calls onEnterApp.
10. **Render UX**: renderImage/animateImage now carry AbortSignal timeouts
    (5 min render / 7 min animate) + a red "Cancel" button in the Visualize
    panel (single render, style grid, Listing Pack, Animate). A stalled GPU
    can no longer hang the UI forever.
11. Small frontend hardening: loadSample re-checks planReqId after its await
    (sample can't clobber a newer upload); keyboard shortcuts ignore SELECT /
    contentEditable (dropdown type-ahead no longer toggles measure/roof);
    prod error copy no longer says "is uvicorn running on :8000?";
    walkthrough recorder stops its captureStream tracks (leak).
12. Stale test updated: test_sample_plan roof/wframe expectations were from
    the pre-HipRoof design and never passed at deb56aa; now pins "GLB ships
    NO roof mesh" (frontend owns the roof).
13. requirements.txt torch==2.13.0 is VALID (exists on PyPI) — audit item
    withdrawn, no change.
